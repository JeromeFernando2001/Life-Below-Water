Technical Description

Navbar & Template

The navigation bar was implemented using an <iframe> to dynamically load different content sections. The navigation links update the iframe source dynamically, which improves performance and prevents full page reloads.

To enhance responsiveness, a **burger menu** was added, which collapses into a dropdown on smaller screens. CSS media queries handle the layout adjustments, ensuring smooth navigation across devices.

Home Page

The **home page** was structured using **flexbox containers and cards** to display content efficiently. Each card contains images, text, and buttons, styled using CSS Grid to ensure a responsive design.

Navigation within the home page is implemented using JavaScript onClick events, which dynamically load content into the iframe without refreshing the page.

Table Page

The table page was structured using the **HTML table element**, with additional features like:

  • **Merged cells** using colspan and rowspan for better readability.
  • **Icons and links** for interactive elements.
  • **Hover effects and alternating row colors** using CSS.

The table design follows accessibility principles, ensuring good contrast, proper padding, and scalable font sizes.

Challenges & Lessons Learned

1. Team Collaboration: Coordinating with multiple team members was challenging due to different coding styles. We overcame this by following a **consistent naming convention** and using comments for clarity.

2. Dynamic Content Loading: Initially, we used traditional navigation but switched to **iframe-based dynamic loading** to enhance performance. This required restructuring our navigation logic.

3. Responsive Design Issues: The original navbar was not fully responsive. We implemented **CSS media queries** and a **burger menu** to resolve this.

Accessibility

Accessibility was a priority, and we followed **WCAG guidelines** by:

  • Using **semantic HTML** elements (<nav>, <main>, <section>, etc.).
  • Ensuring proper **color contrast** for readability.
  • Adding **ARIA labels** for assistive technologies.
  • Supporting **keyboard navigation** for all interactive elements.

Compliance

Our development followed best practices for **security, ethical development, and copyright compliance**, including:

  • Using **HTTPS** to ensure secure connections.
  • Referencing external resources with proper citations.
  • Following **clean coding** principles to maintain readability.
  • Ensuring **inclusive design** to accommodate all users.
