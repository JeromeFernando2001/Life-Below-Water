Profile Page

Technical Description

I have designed the profile page to showcase personal details, skills, goals, and volunteering availability while aligning with the Sustainable Development Goal (SDG) theme: Life Below Water. The HTML structure is well-organized with semantic elements like <section>, <div>, and <h1> to improve readability and maintainability. Each section focuses on distinct content:

Profile Section: Displays user information, including name, bio, and a profile image. The btn-contact link allows users to connect directly.

About Section: Lists personal details and skills, utilizing progress bars to visually represent skill levels.

Goals Section: Utilizes cards to present goals, frustrations, and experiences with clear labels and images.

Calendar Section: Features a dynamic calendar integrated via JavaScript for users to select dates and view event details.

UN Goals Section: Displays SDG-related images with descriptive alt text for improved context.

Contact Form Section: Includes labeled input fields with required attributes to ensure users provide essential information before submitting.

Accessibility

This profile page is designed to be usable by people with disabilities by following these key principles:

Easy Navigation for Screen Readers: The page uses correct HTML heading tags (like <h1>, <h2>, etc.) to organize content, making it easy for screen readers to understand the structure and jump to different sections.

Image Descriptions: Every image has a text description (using the "alt" attribute) that explains what the image is, helping people who can't see the image understand its meaning.

Mouse-Free Navigation: The calendar controls (previous/next buttons) can be used using only the keyboard.

Clear Form Fields: Form labels are correctly linked to their input fields, which helps assistive technology users understand what information is required in each field.

Readable Colors: The colors used for text and progress bars have enough contrast against the background, making them easier to read for everyone, including people with low vision. Visible Focus: When you navigate the page with the keyboard, the currently selected link or button has a clear visual highlight, so you know where you are.

With these practices, the profile page effectively balances design aesthetics with functionality and accessibility, ensuring a positive user experience for all visitors.

Sitemap Page

Technical Description

The sitemap page is designed using SVG elements to create an interactive visual representation of the website's structure. The SVG integrates clickable nodes wrapped in <a> tags, which link to key sections such as profile, volunteer opportunities, reviews, statistics and feedback form. Each clickable node is defined using <rect> elements styled with specific dimensions, rounded corners (rx), and color fills to distinguish different sections. The nodes also include associated <text> tags for clear labeling.

To improve maintainability and flexibility, <path> elements previously used for text have been replaced with <text> tags, allowing better control over styling through CSS. The overall layout is structured to ensure clarity and visual hierarchy, with connecting lines created using SVG elements to represent relationships between different sections.

Accessibility

Accessibility enhancements have been incorporated to ensure the sitemap is navigable and user-friendly for all users, including those relying on assistive technologies. Each clickable <a> tag now includes tabindex="0" to support keyboard navigation and role="link" to improve screen reader compatibility. Additionally, <title> tags have been added within each link to provide meaningful descriptions for screen readers. By replacing text-rendered <path> elements with proper <text> tags, the content is now more accessible, ensuring users can easily interpret the sitemap's structure. These improvements enhance usability for keyboard-only users and those using screen readers, aligning with best practices in web accessibility.

Content Page

Technical Description

The content page is structured using HTML5 with semantic tags for improved organization and readability. The <section> elements are effectively utilized to group content into meaningful parts, such as "What is Overfishing?" and "The Human Cost of Overfishing." Each section is accompanied by descriptive headings (<h1> and <h2>) and paragraphs that provide informative content. External CSS stylesheets are linked for design consistency, with one stylesheet dedicated to content-specific styling and another for global styling. Navigation is enhanced by a sidebar containing anchor links (<a href="#sectionX">) that allow users to jump to relevant content sections. Images are organized in containers with appropriate <img> tags to support visual storytelling.

Accessibility

The page incorporates various accessibility features to enhance usability. The lang="en" attribute is included in the <html> tag to define the language, supporting screen readers. Images are accompanied by meaningful alt text that describes their content, ensuring visually impaired users can understand the visual elements. Clear and informative headings are used throughout the content to improve readability and support keyboard navigation. The sidebar navigation enables quick access to key sections, improving the user experience for those using assistive technologies. While the content is generally accessible, adding features like ARIA landmarks for better screen reader navigation and ensuring color contrast meets accessibility standards would further enhance inclusivity.
